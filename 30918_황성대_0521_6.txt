데이터 정의어(DDL) : 객체를 새로 만들거나 기존에 존재하던 객체를 생성, 변경, 삭제하는 등의 기능을 수행하는 명령어를 데이터 정의어라 한다.

create : 테이블 생성
alter : 이미 생성된 객체를 변경할 때 사용
drop : 객체를 삭제하는 명령

create문
형식:
create table 소유계정, 테이블이름(
열1 이름 자료형,
열2 이름 자료형,
열3 이름 자료형,
```
열n 이름 자료형);

정수 타입 NUMBER(3)
문자 VARCHAR(5)

테이블 이름 생성 규칙
1. 테이블 이름은 문자로 시작해야 한다. (한글도 가능하며 숫자로 시작할 수 없음)
2. 테이블 이름은 30byte 이하여야 한다. (즉 영어30자, 한글은 15자까지 사용가능)
3. 같은 사용자 소유의 테이블 이름은 중복될 수 없다.
4. 테이블 이름은 영문자(한글가능), 숫자(0~9)와 특수문자를 사용할 수 있다.
5. SQL 키워드는 테이블 이름으로 사용할 수 없다.(함수 이름 SELECT, FROM 등은 테이블 이름으로 사용 불가)

열 이름 생성 규치
1. 열 이름은 문자로 시작해야 한다.
2. 열 이름은 30byte 이하여야 한다.
3. 한 테이블의 열 이름은 중복될 수 없다.
4. 열 이름은 열문자(한글가능), 숫자(0~9)와 특수문자를 사용할 수 있다.
5. SQL 키워드는 열 이름으로 사용할 수 없다. (함수 이름 SELECT, FROM 등은 열 이름으로 사용 불가)
